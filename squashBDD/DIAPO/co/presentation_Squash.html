Présentation de Squash

La suite Squash (Software QUality ASsurance enHancement) est un projet Open Source initié par HENIX, qui a pour objet la structuration et l'industrialisation des tests fonctionnels.

Le périmètre du projet Squash recouvre différents domaines de la qualification logicielle

Zoom

Les différents écrans du référentiel de test Squash

Zoom

L'espace administration

Zoom

Cet espace est accessible par l'administrateur uniquement. Il donne (entre-autres) accès aux fonctionnalités d'administration des :

  • Utilisateurs

  • Projets

  • Serveurs d'automatisation

  • Serveurs de code source partagé

  • etc ...

L'espace exigence

Chaque exigence de test décrit un comportement attendu :

  • Dans une gestion de projet traditionnel, une exigence se rédige sous la forme « L'application doit permettre [Action] ».

  • En contexte agile, les User Stories remplacent les exigences (on peut les synchroniser avec un outil de gestion de projet type Jira)

Zoom

L'espace cas de test

Zoom

L'espace Cas de test permet de gérer les cas de test, leurs jeux de données ainsi que la liaison entre cas de test et exigences. Selon l'approche du projet, on pourra créer :

  • des cas de test classiques

  • des cas de test Gherkin

L'espace campagne

L'espace Campagnes permet la gestion des campagnes de tests, l'exécution des tests et la remontée des anomalies.

Si les tests ont été automatisés, on pourra commander leur exécution depuis cet espace.

Zoom

L'espace automatisation

Zoom

L'espace automatisation sert à la gestion et au suivi des cas de tests éligibles à l'automatisation. Il offre deux écrans différents selon le profil de l'utilisateur connecté : Automaticien OU Tester

Les cas de tests présents se trouvent à des statuts différents selon leur état d'avancement dans le workflow d'automatisation :

  • En cours de rédaction

  • Prêt pour transmission

  • Transmis

  • Rejeté

  • En cours d'automatisation

  • Suspendu

  • Automatisé

Manipuler le référentiel de test Squash

  • Lancez Squash

  • Connectez vous en tant qu'administrateur (admin/admin)

  • Repérez les différents espaces (exigence, cas de test, campagne, automatisation, administration)

  • Faites l'inventaire du patrimoine existant
