Les outils du BDD - langage Gherkin et framework d'implémentation

Le langage Gherkin

Gherkin est le langage dédié à l'écriture de scénario BDD en vue d'une automatisation. Loin de l'écriture des tests fonctionnels en cycle en V (couple action / résultat attendu), les scenarii de test BDD se calquent sur le formalisme des User Stories. Les fonctionnalités sont décrites du point de vue de l'utilisateur plutôt que du point du vue logiciel.

Écrire une User Story - la description

Une User Story comprendra d'abord une DESCRIPTION avec les mots clés suivants :

AS En tant que ‘un utilisateur'

I WANT TO Je veux ‘une action'

IN ORDER TO Afin de ‘But de la story'

En tant que <rôle>, je veux pouvoir <action>, afin de <objectif>.

Ex : "En tant que client de la banque, je veux pouvoir me connecter à mon espace en ligne pour accéder à mon compte"

Écrire une User Story - les critères d'acceptation

Les critères d'acceptation constituent les scenarii permettant de définir un ensemble de conditions que la story doit satisfaire pour être considérée comme complète et terminée.

Pour formaliser les critères d'acceptation, on utilise souvent le langage Gherkin et la structure GIVEN-WHEN-THEN.

GIVEN : Qui décrit l'état initial de l'application ( Etant donné que ‘contexte' => Prérequis du cas de test)

WHEN : Qui décrit un événement ou un moment précis (Quand ‘un événement' => Action du cas de test)

THEN : Qui précise la réaction du SI à l'événement décrit dans le WHEN (Alors ‘un certain résultat' => Résultat attendu du cas de test)

GIVEN Je suis un client de la banque

WHEN je me connecte au site avec mon login et un mot-de-passe valides

THEN j'accède à mon espace personnel

Parce qu'ils s'écrivent avec des mot clefs spécifiques, les scripts Gherkin respectent un formalisme qui facilite leur automatisation.

Automatiser les tests

Des spécifications exécutables

Les frameworks d'implémentation BDD servent à automatiser l'exécution d'un scénario de test. Ils agissent comme des parseurs qui interprètent les mots-clés Ghekin (GIVEN, WHEN, THEN) comme des méthodes (fonctions) que le développeur ou automaticien de test doit implémenter.

Les frameworks d'implémentation

Le langage de spécification des critères d'acceptation de la User Stories (le Gherkin) étant agnostique, l'implémentation des méthodes peut être réalisée dans n'importe quel langage de programmation.

  • Cucumber (Java, Ruby, JavaScript) et ses extensions : Behave (Python), CukeSalad (orienté ATDD : task, goal, action)

  • Jbehave (Java)

  • Turnip (Ruby)

  • Spinach

  • ...
