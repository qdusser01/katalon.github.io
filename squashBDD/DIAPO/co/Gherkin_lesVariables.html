La gestion des données de test à partir d'un script Gherkin

Il est important de distinguer au sein d'un script Gherkin ce qui est de l'ordre de la spécification invariable et ce qui est de l'ordre de la donné de test.

Donnée de test

Donnée existante (dans une base de données, dans un tableau...) avant qu'un test ne soit exécuté et qui affecte ou est affectée par le SUT. Les données de test sont le résultats de l'expertise du testeur, qui les définit par des techniques de conception tels que les partitions d'équivalence.

Gherkin offre différentes manières de traiter ces données.

Les tables de données

Les données de test peuvent être spécifiées dans des tables de données.

Zoom

Les chaînes de caractères et les caractères numériques

Par défaut, un éditeur Gherkin signale toutes valeurs numériques ou chaîne de caractère entre guillemet à la manière d'une variable (colorimétrie)

Zoom

Les plans de scénario avec paramétrage du script de test

Basé sur le principe du test piloté par les données, Gherkin offre la possibilité de répéter l'exécution d'un même script autant de fois que l'ont dispose de données de test.

Pour écrire un tel script, on utilisera :

  • Le mot-clé Scenario Outline (en français : Plan du Scénario)

  • Les chevrons <> pour définir les paramètres de test

  • Le mot-clé Examples (en français : Exemples)sous lequel la table de données spécifie les valeurs des paramètres pour chaque test.
