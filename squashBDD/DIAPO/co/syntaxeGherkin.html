La syntaxe Gherkin dans Squash

La syntaxe d'écriture Gherkin dans Squash se calque sur les conventions du Gherkin traditionnel.

Zoom

Gherkin n'est pas un langage de programmation, mais un Domain Specific Language (DSL), basé sur les mots-clés, dont voici les règles de base.

1°. Un cas de test Gherkin teste UNE SEULE fonctionnalité

Un scipt Gherkin correspond à une fonctionnalité. C'est d'ailleurs par ce mot-clé que commence le script :

En français : "Fonctionnalité"

En anglais : "Feature"

2°. Une fonctionnalité se vérifie par UN ou PLUSIEURS scénarios

Un scipt Gherkin est constitué de 1 à n scénarios. Ces scénarios sont autant de tests visant à valider la fonctionnalité.

En français : "Scénario"

En anglais : "Scenario"

3°. Un scénario DOIT contenir les trois mots clés GIVEN, THEN, WHEN

Given, When, Then constituent les trois étapes obligatoire d'un scénario Gherkin

GIVEN : décrit l'état initial de l'application ( Etant donné que ‘contexte' => Prérequis du cas de test)

WHEN : décrit un événement ou un moment précis (Quand ‘un événement' => Action du cas de test)

THEN : précise la réaction du SI à l'événement décrit dans le WHEN (Alors ‘un certain résultat' => Résultat attendu du cas de test)

Ces mots-clés peuvent se répéter selon si l'étape se compose d'une ou plusieurs conditions, actions ou vérifications.
