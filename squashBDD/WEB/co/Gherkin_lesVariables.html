La gestion des données de test à partir d'un script Gherkin

Il est important de distinguer au sein d'un script Gherkin ce qui est de l'ordre de la spécification invariable et ce qui est de l'ordre de la donné de test.

Définition : Donnée de test

Donnée existante (dans une base de données, dans un tableau...) avant qu'un test ne soit exécuté et qui affecte ou est affectée par le SUT. Les données de test sont le résultats de l'expertise du testeur, qui les définit par des techniques de conception tels que les partitions d'équivalence[1].

Gherkin offre différentes manières de traiter ces données.

Les tables de données

Les données de test peuvent être spécifiées dans des tables de données.

Les chaînes de caractères et les caractères numériques

Par défaut, un éditeur Gherkin signale toutes valeurs numériques ou chaîne de caractère entre guillemet à la manière d'une variable (colorimétrie)

Les plans de scénario avec paramétrage du script de test

Basé sur le principe du test piloté par les données[2], Gherkin offre la possibilité de répéter l'exécution d'un même script autant de fois que l'ont dispose de données de test.

Pour écrire un tel script, on utilisera :

  • Le mot-clé Scenario Outline (en français : Plan du Scénario)

  • Les chevrons <> pour définir les paramètres de test

  • Le mot-clé Examples (en français : Exemples)sous lequel la table de données spécifie les valeurs des paramètres pour chaque test.

Notes

  1. Partition d'équivalence

    La technique des partitions d'équivalence est utilisée pour réduire le nombre de cas de test nécessaire pour tester un ensemble d'entrées, de sorties, d'intervalles de valeurs ou de temps.

    Le découpage en partitions est utilisé pour créer des classes d'équivalence (souvent appelées partitions d'équivalence) qui sont constituées d'ensembles de valeurs qui sont traités de la même manière.

    En sélectionnant dans chaque partition une valeur représentative, la couverture de tous les éléments de la même partition est assurée

    ex : la carte de réduction de la SNCF "12-25" implique trois partition d'équivalence :

    [0 ; 11] invalide

    [12 ; 25] valide

    [26 ; +INF] invalide

  2. test piloté par les données

    Une technique de script qui sauvegarde les entrées de tests et les résultats attendus dans une table ou un tableur, de façon à ce qu'un simple script de contrôle puisse exécuter tous les tests de la table.
