Vers l'automatisation des test

Rappel :

Les tests Gherkin ont vocation à être automatisés. Cela implique qu'à chaque mot-clé soit rattaché du code (on parle d'implémentation).

L'implémentation d'un test

L'implémentation est le travail de l'automaticien. Ce dernier utilise un framework (ex : Cucumber) capable :

  • d'interpréter le langage Gherkin

  • de faire le lien entre un script Gherkin et les méthodes implémentées

  • d'exécuter un script Gherkin (les actions et vérification automatisées)

  • de générer un rapport d'exécution

Comprendre l'interprétation des mots-clés par Cucumber

Les choix d'écriture du script dirigent le travail d'implémentation. C'est pourquoi, un testeur fonctionnel doit être en mesure de comprendre et de prédire l'interprétation de son script par le framework Cucumber.

Nous allons voir dans le détail comment s'articule l'interprétation de Gherkin par Cucumber.
