Exercice synchronisation

Le rejeu d'un scénario peut facilement partir en échec pour cause de synchronisation. Ex. : l'automate tente de cliquer sur un élément web avant que sa page ait fini de charger. Les logs présenteront alors une erreur du type : "Element not fond"

Pour prévenir ces échecs techniques, il est possible d'ajouter des pas de synchronisation :

  • Wait for element visible

  • Wait for alert

  • etc...

Question

Éditez votre script de manière à gérer la synchronisation à chaque changement de page
