Les Systèmes Sous Test

Zoom

System Under Test (SUT) : En français, système sous test, définit les applications / modules / composants faisant partie du périmètre à tester.

Applications à Tester

Batches (traitement par lot)

Services web (protocole d'interface et de communication inter-applicatif)

IHM web (accès via un navigateur web, éventuellement enrichi de plug-ins)

  • Applications web « traditionnelles » (clients « légers »)

  • Applications Internet Riches (clients « riches »)

    • Interfaces se rapprochant de celles des clients lourd

    • Traitements côté client (utilisation de JavaScript, Ajax, ActiveX, Java, Adobe Flash, Silverlight, etc.)

Autres IHM (applications en « standalone », clients « lourds », émulateurs de terminaux mainframe, etc.)

Données à vérifier

  • Bases de données relationnelles

  • Fichiers (fichiers XML, fichiers séquentiels à champs fixes, fichiers séquentiels à champs variables, etc.)
